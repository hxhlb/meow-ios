Build 2026051810 (1.3.1)

WHAT'S NEW
• mihomo-rust engine bumped 0.7.3 → 0.7.6. Two consequential fixes
  inside that bump:
  – ECH (Encrypted Client Hello) bootstrap now succeeds on iOS NE,
    where it was silently failing in 1.3.0. iOS's bootstrap UDP/53
    DNS path was returning a 5-byte ECH GREASE stub instead of the
    real ECHConfigList; BoringSSL then rejected the dial with
    INVALID_ECH_CONFIG_LIST and any proxy with `ech-opts:` couldn't
    handshake. v0.7.6 validates the SvcParam #5 shape and falls back
    to DoH against 1.1.1.1 / 8.8.8.8 over TCP+TLS, so the lookup
    can't be DNS-hijacked. The practical effect: SNI is no longer
    sent in cleartext to ECH-protected proxies, materially reducing
    the chance of upstream server-IP blocks.
  – Per-flow footprint cap: mihomo-tunnel now bounds inbound
    connection count and WebSocket buffers, complementing the iOS
    NE accept-cap. Helps keep the PacketTunnel extension under
    iOS's ~50 MB jetsam ceiling on long-running heavy sessions.

• BoringSSL backend enabled via the `boring-tls` cargo feature.
  Required to make the new ECH path work; also activates real uTLS
  fingerprinting (`client-fingerprint: chrome` etc.) instead of the
  prior stub-warn behavior. PacketTunnel.appex size grew ~1 MB on
  disk (binary, not runtime memory).

• Subscription fetch now sends `User-Agent: clash.meta/<version>`
  matching the embedded engine, so subscription panels serve the
  full SS/Trojan/VLESS proxy set instead of CN-bypass-only stubs.

• PacketTunnel extension now self-restarts when physical memory
  footprint crosses 40 MB. iOS jetsam-kills NE extensions silently
  around 50 MB; the watchdog turns that into an orderly restart
  with a logged reason and a state-row breadcrumb for the app to
  surface.

• Memory-investigation pass under sustained TCP churn: ruled out
  accept-cap, DNS scheme, and proxy outbound as the dominant
  growth lever. The remaining ~4 KiB-per-accepted-flow growth is
  documented in `docs/INVESTIGATION-2026-05-16-stress-rss-netstack-smoltcp.md`
  for follow-up; v0.7.6's footprint cap is the first iteration of
  the upstream fix.

KNOWN
• Non-DNS UDP forwarding still pending (QUIC/HTTP3 degraded to TCP).

FOR TESTERS
• Pay particular attention to ECH-protected proxies — sites that
  used to fail on 1.3.0 with ECH-tagged entries should now work.
  If you previously saw "connected but no internet" with the
  current subscription, retry on 1.3.1.
• Long-running sessions: leave the VPN connected through a few
  hours of heavy browsing and confirm the extension doesn't drop.
  If it does, please report the build number + how long it ran
  before the drop.
